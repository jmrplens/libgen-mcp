libgen-mcp ships a **gated, LLM-driven evaluation harness** (`cmd/eval`) that goes
beyond unit and end-to-end tests: it connects a **real language model** (Claude
Haiku) to a **real libgen-mcp server** over the MCP protocol and checks that the
model actually _uses the tools correctly_ against the **live** Library Genesis
site — the right tool, the right arguments, sensible recovery on failure, and
correct handling of the structured output.

## How it works

For each scenario the harness:

1. Applies any per-scenario environment, then builds a fresh in-process
   libgen-mcp server with the four tools registered.
2. Sends the scenario's natural-language prompt plus the tool definitions to the
   model (up to six turns).
3. Executes every tool call the model makes **against the real server** (real
   mirrors, real downloads) and feeds the results back.
4. Grades the transcript: the tool name, the argument shape, and whether the
   real MCP response is well-formed — never exact catalog content, which drifts.

Each scenario returns **PASS**, **FAIL**, or **SKIP**. A SKIP means the model did
the right thing but a live mirror was unavailable (e.g. a rotating Sci-Hub host or
a fresh-mirror miss) — a network condition, never a model failure.

<Aside type="note">
	The harness is behind the `eval` build tag and only runs with `LIBGEN_EVAL=1`
	and an `ANTHROPIC_API_KEY`. It spends real API tokens and performs real
	downloads, so it never runs in CI — it is an on-demand quality check.
</Aside>

## Scenarios

Forty-one scenarios cover search, details, downloads, source selection, error
recovery, progress, link resolution, reading and summarizing extracted text,
open-access discovery, the escalation beyond the Library Genesis catalog, and —
importantly — **unguided** prompts that give the model no hints, testing whether
the tool surface is self-describing enough for an LLM to use unaided.

S21–S31 cover the capabilities shipped since v1.2.0: BibTeX/RIS **citations**,
opt-in **enrichment** (Crossref journal and citation-count metadata), the `read`
tool's **find** (in-document search) and **outline** (table of contents) modes,
and capability-gated **elicitation** (an on-demand Unpaywall email prompt, and a
download-confirmation prompt) with its deterministic fallback.

S32–S41 cover the **extra search sources**: a title the catalog does not carry
must still be found, described, downloaded and read through Anna's Archive; the
three deployment modes must behave as documented, including `never` being a lock
no call can lift; and the membership tier must be discoverable from the tool
surface alone. Several repeat against a server in **remote** (`--http`) mode.

## Local and remote blocks

Most scenarios run against a **local** server, where `download` writes the file to
disk. S17–S18 form a **remote block**: the same download requests run against a
server in remote mode (`--http`), where `download` returns a link instead of
saving a file. The harness then fetches that link to the sandbox — standing in
for an agent's own fetch tool — so the file lands locally in both cases.
Together the two blocks verify the model behaves the same while the server
behaves differently, and the file ends up local either way. S27–S31, and the
remote half of S32–S37, form a second remote block: the read, open-access,
citation, enrichment and escalation checks repeated over the same `--http`
transport, ten scenarios in all.

`claude-haiku-4-5` (real Anthropic API, real mirrors, real downloads):

The three skips are live-data conditions (an HTTP 403, a scanned PDF with no text
layer, a file whose container the extractor does not support) — never a model or
server failure. The one failure, S29, is a genuine model-behavior finding rather
than a server bug: the model did set `extra_sources` and did receive open-access
hits, but its answer named no identifier from them. It is left red on purpose; a
harness that measures how well a model uses the tools should not be tuned until
it agrees with itself.

## Running it yourself

```sh
# All scenarios (real API + mirrors + downloads)
LIBGEN_EVAL=1 ANTHROPIC_API_KEY=… make eval

# A subset, and keep the downloaded files
LIBGEN_EVAL=1 ANTHROPIC_API_KEY=… go run -tags eval ./cmd/eval --only S1,S9,S14 --keep-downloads
```

The process exits non-zero if any scenario **fails** or **errors** (SKIPs and
PASSes do not). See [`cmd/eval/README.md`](https://github.com/jmrplens/libgen-mcp/blob/main/cmd/eval/README.md)
for the full harness documentation.
