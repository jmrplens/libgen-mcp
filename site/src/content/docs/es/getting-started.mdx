Esta página te guía para instalar `libgen-mcp`, conectarlo a un cliente MCP y ejecutar tu primera búsqueda.

## Instalación

La instalación recomendada es el **binario precompilado**: un único ejecutable estático sin nada más que instalar — sin toolchain de Go, sin Docker, sin runtime. Docker y `go install` se ofrecen como alternativas.

<Tabs>
	<TabItem label="Binario (recomendado)">
		Descarga un binario precompilado para tu plataforma desde la página de [releases de GitHub](https://github.com/jmrplens/libgen-mcp/releases). Los archivos se llaman `libgen-mcp-<os>-<arch>` (por ejemplo `libgen-mcp-linux-amd64` o `libgen-mcp-darwin-arm64`), cubriendo Linux, macOS y Windows en amd64 y arm64.

    	```bash
    	# Ejemplo: Linux amd64
    	curl -L -o libgen-mcp \
    	  https://github.com/jmrplens/libgen-mcp/releases/latest/download/libgen-mcp-linux-amd64
    	chmod +x libgen-mcp
    	sudo mv libgen-mcp /usr/local/bin/
    	```

    	El binario es totalmente estático (compilado con `CGO_ENABLED=0`), así que no depende de nada en el host y se ejecuta directamente. Cada release incluye también un `checksums.txt`; verifica tu descarga contra él antes de ejecutarla.
    </TabItem>
    <TabItem label="Docker">
    	¿Prefieres contenedores, o un comando sin instalación que tu cliente descarga en el primer arranque? Se publica una imagen multi-arch en el GitHub Container Registry:

    	```bash
    	docker pull ghcr.io/jmrplens/libgen-mcp:latest
    	```

    	La imagen se ejecuta en stdio por defecto; para el transporte HTTP en streaming pasa `--http 0.0.0.0:8080` (y publica el puerto). En modo HTTP se expone además un endpoint `GET /health`:

    	```bash
    	docker run --rm -p 8080:8080 \
    	  -v "$HOME/Downloads:/downloads" \
    	  -e LIBGEN_MCP_DOWNLOAD_DIR=/downloads \
    	  ghcr.io/jmrplens/libgen-mcp:latest --http 0.0.0.0:8080
    	```

    	Monta un volumen con permisos de escritura para las descargas y apunta `LIBGEN_MCP_DOWNLOAD_DIR` a él; el contenedor se ejecuta como usuario sin privilegios, por lo que el directorio del host debe ser escribible por el UID `10001`.
    </TabItem>
    <TabItem label="go install">
    	Si ya tienes Go 1.26 o posterior y prefieres compilar desde el código fuente:

    	```bash
    	go install github.com/jmrplens/libgen-mcp/cmd/server@latest
    	```

    	Esto produce un binario llamado `server` en `$(go env GOPATH)/bin`. Si prefieres invocarlo como `libgen-mcp`, compílalo con un nombre explícito:

    	```bash
    	git clone https://github.com/jmrplens/libgen-mcp
    	cd libgen-mcp
    	go build -o libgen-mcp ./cmd/server
    	```

    	Asegúrate de que el binario resultante esté en tu `PATH`.
    </TabItem>

</Tabs>

## Configurar un cliente MCP

Apunta tu cliente al binario. El comando es `libgen-mcp` (o la ruta absoluta al binario `server` si mantuviste el nombre por defecto de `go install`). Sobre stdio no hacen falta argumentos adicionales.

### Claude Code

Añade el servidor al `.mcp.json` de tu proyecto (o ejecuta `claude mcp add`):

```json
{
	"mcpServers": {
		"libgen": {
			"command": "libgen-mcp"
		}
	}
}
```

### Claude Desktop

La vía más sencilla es la extensión **`.mcpb`** de un solo clic de la última [release](https://github.com/jmrplens/libgen-mcp/releases) (universal para macOS + Windows): descárgala y ábrela con Claude Desktop. Editar a mano `claude_desktop_config.json` es la alternativa.

Edita `claude_desktop_config.json` (`~/Library/Application Support/Claude/` en macOS, `%APPDATA%\Claude\` en Windows) y añade el mismo bloque `mcpServers`, luego reinicia Claude Desktop:

```json
{
	"mcpServers": {
		"libgen": {
			"command": "/absolute/path/to/libgen-mcp",
			"env": {
				"LIBGEN_MCP_DOWNLOAD_DIR": "/absolute/path/to/downloads"
			}
		}
	}
}
```

<Aside type="caution">
	Los clientes de escritorio no heredan el `PATH` de tu shell, así que usa una
	ruta `command` absoluta.
</Aside>

### VS Code

El soporte MCP de VS Code (o las extensiones Continue / Cline) lee un `mcp.json` con la misma forma. En el formato propio de VS Code:

```json
{
	"servers": {
		"libgen": {
			"command": "libgen-mcp",
			"type": "stdio"
		}
	}
}
```

### Remoto (HTTP en streaming)

Para ejecutar el servidor de forma centralizada y conectarte por HTTP en lugar de stdio, inícialo con una dirección y apunta los clientes con capacidad HTTP hacia él:

```bash
libgen-mcp --http :8080
```

En modo HTTP el servidor expone además un endpoint `GET /health` que devuelve `200` cuando está sirviendo, útil para sondas de disponibilidad. Como en este modo el servidor es remoto y no puede escribir en tu disco, la herramienta `download` devuelve siempre un enlace de descarga (en vez de guardar el archivo) — `resolve_only` queda implícito, no hace falta indicarlo.

<Aside type="note">
	Alojar en su lugar un servidor **stdio** — por ejemplo detrás de `mcp-proxy`
	en un catálogo como Glama — también lo coloca en una máquina remota o efímera,
	solo que sin un listener HTTP. Fija `LIBGEN_MCP_REMOTE_DOWNLOADS=1` para poner
	a ese servidor stdio en el mismo modo de descarga remota que `--http` obtiene
	automáticamente.
</Aside>

Consulta [Configuración](/libgen-mcp/es/configuration/) para las variables de entorno que puedes fijar en cualquiera de estos casos, y [Arquitectura](/libgen-mcp/es/architecture/) para saber cómo funcionan los transportes.

## Tu primera búsqueda

<Aside type="caution">
	`libgen-mcp` accede a mirrors de terceros de Library Genesis. Eres responsable
	de respetar las leyes de derechos de autor y de propiedad intelectual que
	apliquen en tu lugar de residencia. Úsalo solo para contenido al que tengas
	derecho legal de acceder.
</Aside>

Cuando el cliente muestre `libgen` como conectado, pídele que busque. Una petición como:

> Busca en Library Genesis "the go programming language" en no ficción, 25 resultados.

acciona la herramienta `search` con más o menos estos argumentos:

```json
{
	"query": "the go programming language",
	"topics": ["nonfiction"],
	"results_per_page": 25
}
```

Cada resultado incluye un `md5` (para libros) o un `doi` (para artículos). Pasa un `md5` a `get_details` para obtener los metadatos completos, y luego a `download` para descargar el archivo — o pasa un `doi` directamente a `download` para un artículo. `get_details` también devuelve un campo `citations` (`bibtex`/`ris`) que puedes pegar directamente en un gestor de referencias. Consulta [Herramientas](/libgen-mcp/es/tools/) para ver todas las formas de entrada y salida.

## Prompts

Además de las tres herramientas, el servidor registra cuatro **prompts** MCP que tu cliente puede ofrecer como acciones rápidas — cada uno convierte una petición habitual en un plan de llamadas a herramientas listo para ejecutar, sin descargar nada por sí mismo:

| Prompt                  | Argumentos                                                       | Qué hace                                                                                     |
| ----------------------- | ---------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| `acquire_book`          | `title` (requerido), `author`, `format`, `language`              | Encuentra y confirma la edición de un libro que mejor coincide, y la descarga.               |
| `research_topic`        | `topic` (requerido), `kind` (`articles`/`books`/`both`), `limit` | Construye una lista de lectura de artículos y/o libros sobre un tema, los descarga y resume. |
| `get_paper`             | uno de `doi` o `citation`                                        | Obtiene un artículo concreto directamente por DOI, o lo busca por una cita en texto libre.   |
| `download_troubleshoot` | `md5`, `doi`, `error` (todos opcionales)                         | Obtiene un árbol de decisión para diagnosticar y recuperarse de una descarga fallida.        |

Consulta [Herramientas](/libgen-mcp/es/tools/#prompts) para ver la tabla completa de argumentos y el comportamiento de cada prompt.

## Preguntas frecuentes

### ¿Cómo instalo libgen-mcp?

Descarga un binario precompilado para tu plataforma desde la página de [releases de GitHub](https://github.com/jmrplens/libgen-mcp/releases) (los archivos se llaman `libgen-mcp-<os>-<arch>`, cubriendo Linux, macOS y Windows en amd64 y arm64), hazlo ejecutable y colócalo en tu `PATH`. Como alternativas puedes usar la imagen de Docker multi-arch `ghcr.io/jmrplens/libgen-mcp` o, si ya tienes Go 1.26 o posterior, compilar desde el código fuente con `go install github.com/jmrplens/libgen-mcp/cmd/server@latest`.

### ¿Qué método de instalación debería usar?

El binario precompilado es la vía recomendada: es un único ejecutable estático sin nada más que instalar — sin toolchain de Go, sin Docker, sin runtime. Elige Docker si prefieres contenedores o un comando sin instalación que tu cliente descarga en el primer arranque, y `go install` si ya tienes el toolchain de Go y prefieres compilar desde el código fuente.

### ¿Cómo conecto libgen-mcp a Claude o VS Code?

Apunta tu cliente MCP al binario `libgen-mcp` por stdio, sin argumentos adicionales. En Claude Code añade el servidor a `.mcp.json` (o ejecuta `claude mcp add`); en Claude Desktop usa la extensión `.mcpb` de un clic o añade un bloque `mcpServers` a `claude_desktop_config.json` con la ruta absoluta del binario; en VS Code añade un bloque `servers` a `mcp.json` con `type` stdio. Para ejecutarlo de forma centralizada, inícialo con `libgen-mcp --http :8080` y conecta hacia él clientes con capacidad HTTP.

### ¿libgen-mcp necesita un token o cuenta?

No. libgen-mcp no necesita ningún token, cuenta ni clave de API para funcionar: apuntas tu cliente al binario y ya está. Todas las variables de entorno son opcionales y tienen un valor por defecto documentado.
