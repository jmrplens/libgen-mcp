# Fase 2 — Auditoría de tests (unit + e2e) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Cobertura exhaustiva: unit tests con fixtures para las 7 categorías, todas las queries/opciones/opciones-de-descarga, las 3 tools y TODAS las rutas de error (incluidas las nuevas de robustez de Fase 1); y una suite e2e contra el sitio real, gated (skip si no responde), nightly + manual.

**Architecture:** Fixtures reales capturadas una vez en `internal/libgen/testdata/`. Suite e2e en `test/e2e/` con build tag. CI: gate de cobertura a 85% + workflow e2e nocturno + check de compilación e2e en el CI normal.

**Tech Stack:** Go test + httptest, `gotest.tools/gotestsum` (CI e2e), build tags.

## Global Constraints

- **Fase 1 debe estar completa y mergeada** (esta fase testea su superficie).
- Unit tests NUNCA tocan red (fixtures + httptest). E2E SÍ toca red pero solo bajo build tag `e2e` + `LIBGEN_E2E=1` + sonda de alcanzabilidad; nunca falla por sitio caído (usa `t.Skip`).
- E2E asere **estructura, no valores exactos** (conteos/títulos/orden derivan). MD5 con regex `^[a-f0-9]{32}$`.
- Reutilizar `staticMirrors`/`newTestClient`. Fixtures capturadas de `libgen.li` (o de otro mirror de la familia si falla), espaciadas ~1s (buen ciudadano).
- `golangci-lint`/`govulncheck` a cero; `gofmt`/`vet` limpios. Commits path-scoped con trailer `Claude-Session`.
